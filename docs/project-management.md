# Project Management

## Organización del proyecto

Para la gestión del desarrollo de HabitFlow se utiliza un tablero de Trello basado en la metodología Kanban. Este enfoque permite visualizar el estado de cada tarea y organizar el flujo de trabajo de forma clara.

El tablero está dividido en las siguientes columnas:

- **Backlog**: Contiene todas las tareas del proyecto aún no iniciadas.
- **Todo**: Tareas que se van a abordar próximamente.
- **In Progress**: Tareas en desarrollo.
- **Review**: Tareas finalizadas que están siendo revisadas.
- **Done**: Tareas completamente finalizadas.

## Gestión de tareas

Cada funcionalidad del proyecto se representa como una tarjeta dentro del tablero. Estas tarjetas incluyen checklists que dividen la funcionalidad en subtareas técnicas, facilitando el desarrollo paso a paso.

Las tarjetas se mueven entre columnas según su estado, permitiendo tener una visión clara del progreso del proyecto en todo momento.

## Metodología utilizada

Se ha optado por una organización basada en Kanban, ya que permite una gestión flexible del trabajo y se adapta bien a proyectos individuales. Este enfoque facilita priorizar tareas, detectar bloqueos y mantener un flujo de trabajo continuo.

Además, este sistema se complementa con los principios de Agile, permitiendo adaptar el desarrollo según las necesidades que vayan surgiendo durante el proyecto.
